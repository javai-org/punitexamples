# Generated Specs

This directory holds specs generated by PUnit measure experiments. The YAML
files here are gitignored because they regenerate on every run, but this is
simply because punitexamples is designed for pedagogic purposes. A real project
normally commits measure experiment specs as they are required for probabilistic
test runs in CI and (in sentinel applications) in production. See
`../specs-reference/` for committed examples of what these files look like.
